AMB / Concurrent Graph Engine

CSR-optimized graph processing for parallel BFS and PageRank on multi-core CPUs.

This project focuses on the real systems problem behind graph analytics: pointer-heavy layouts and shared-state locking collapse under skewed graphs. AMB uses compressed sparse row storage, atomic update paths, and benchmark comparisons to show how memory layout and synchronization strategy affect throughput, contention, and scalability.

How To Use This Demo

  1. Pick an algorithm. `Parallel BFS` lights up the graph. `PageRank` does not animate yet.
  2. Pick how many threads you want to use.
  3. If you use BFS, keep the source vertex as `0`.
  4. Click `Run Demo`.
  5. Look at the picture for BFS, the log on the right, and the benchmark card below.

Simple rule: BFS shows movement. PageRank shows numbers and benchmark results.

Metric Definitions

TEPS
Traversed Edges Per Second. Higher means the engine is processing more edges each second.
Elapsed
Total runtime for the workload. Lower is better.
Sync Ratio
Fraction of hot-path time spent coordinating threads with atomics or locks.
Est. LLC Miss
Estimated cache-miss pressure. Lower usually means the memory layout is more cache-friendly.
Bandwidth
Approximate memory throughput used while traversing graph data.
